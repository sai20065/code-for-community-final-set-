# Firestore data schema

One table per collection, generated from the model files listed under each
heading so this can't silently drift from what's actually persisted. If you
add or rename a field, update the model file first, then this table.

## `users/{uid}` — `lib/core/models/user_model.dart`

| Field | Type | Meaning |
|---|---|---|
| `name` | string? | Citizen's name — typed or Aadhaar-OCR'd |
| `age` | int? | From Basic Info step |
| `pincodeHome` | string? | Home pincode — typed or Aadhaar-OCR'd |
| `addressHome` | string? | Home address — typed or Aadhaar-OCR'd |
| `wardNumber` | string? | Optional; local ward, typed or Aadhaar-OCR'd |
| `location.lat` / `location.lng` | number? | Home coordinates (Location Setup) |
| `role` | string | `"citizen"` or `"official"` |
| `constituencyId` | string? | Resolved from pincode in Location Setup |
| `homeBoothId` / `homeBoothName` | string? | Nearest booth, resolved alongside constituency |
| `preferredLanguage` | string | ISO language code, default `"en"` |
| `signInMethod` | string? | `"anonymous"` \| `"phone"` \| `"email"` — how this citizen authenticated |
| `aadhaarExtractionConfidence` | number? | 0–1 confidence score from the last Aadhaar OCR run, if any |
| `createdAt` | timestamp | When `users/{uid}` was first written |
| `signupCompletedAt` | timestamp? | Set once Sign Up finishes — the authoritative "this is a real, saved profile" marker; Sign In and Splash routing trust this over any local per-device state |

## `submissions/{id}` — `lib/core/models/submission_model.dart`

| Field | Type | Meaning |
|---|---|---|
| `userId` | string | Author's uid |
| `type` | string | `voice` \| `text` \| `photo` \| `video` |
| `submissionCategory` | string | `problem` (civic issue) or `feedback` (development suggestion) |
| `inputMode` | string | How it was composed (mirrors `type` today) |
| `rawText` | string? | Typed text, or Gemini's audio transcript |
| `mediaUrl` | string? | Cloud Storage download URL for voice/photo/video |
| `transcript` | string? | Gemini transcription of voice media, original language |
| `translatedText` | string? | Cloud Translate output, English |
| `language` | string | Citizen's language for this ticket |
| `theme` | string? | One of 6 categories — citizen-picked or Gemini-classified |
| `clusterId` | string? | Which `clusters/{id}` this ticket was grouped into |
| `priorityScore` | number? | Hint carried from classification, distinct from the cluster's own score |
| `location.pincode` / `.lat` / `.lng` / `.boothId` / `.constituencyId` | mixed | Where the ticket was filed |
| `status` | string | `new` \| `reviewed` \| `inProgress` \| `resolved` |
| `tokenId` | string | Structured receipt id — see pattern below |
| `createdAt` / `updatedAt` / `resolvedAt` | timestamp | Lifecycle timestamps |
| `supporterCount` | int | Citizens backing a `feedback` ticket |
| `supporterIds` | string[] | Uids who've toggled support (dedup guard) |

**Token id pattern:** `PD-<CONSTITUENCY_CODE>-<CATEGORY_CODE>-<YYMMDD>-<SEQ4>`,
e.g. `PD-BLRN-PRB-260706-0007` — constituency (first 4 letters), `PRB`
(problem) or `SUG` (suggestion), creation date, and an atomic same-day
sequence number. Generated by `FirestoreService.generateTokenId`.

## `clusters/{id}` — `lib/core/models/cluster_model.dart`

| Field | Type | Meaning |
|---|---|---|
| `constituencyId` / `boothId` | string | Scope of this recurring issue |
| `theme` | string | One of the 6 category ids |
| `centroidVector` | number[] | Reserved for future semantic clustering — always `[]` today; grouping is by exact (constituency, theme, booth) match |
| `submissionCount` | int | Tickets grouped into this cluster |
| `sampleSubmissionIds` | string[] | A few member ticket ids, for Gemini's summary prompt |
| `summaryText` | string | Gemini-written one-line summary for the MP dashboard |
| `priorityScore` | number? | Gemini-scored 1–100, refreshed every 5th ticket |
| `title` / `localContext` / `affectedBoothRange` | string? | Only populated by the demo seed script today — not yet written by the live pipeline |
| `demandScore` / `demographicScore` / `infraGapScore` | number? | Same caveat — ranked-works/compare-proposals screens show these empty for organically-created clusters |

## `booths/{id}` — `lib/core/models/booth_model.dart`

| Field | Type | Meaning |
|---|---|---|
| `constituencyId` | string | Parent constituency |
| `name` | string | Booth name |
| `lat` / `lng` | number | Map marker position |
| `pincodesCovered` | string[] | Pincodes resolving to this booth |
| `openIssueCount` | int | Drives the red/amber/green density marker |
| `submissionVolume` | int | Drives marker size on the official map |
| `dominantTheme` / `localContext` | string? | Shown in the booth detail sheet |

## `constituencies/{id}` — `lib/core/models/constituency_model.dart`

| Field | Type | Meaning |
|---|---|---|
| `name` / `state` | string | Display name |
| `mpUserId` | string | Official account's uid for this seat |
| `boundaryGeoJson` | map? | Reserved for boundary overlays — not currently rendered |

## `counters/{constituencyId}_{yyMMdd}`

| Field | Type | Meaning |
|---|---|---|
| `count` | int | Atomic per-constituency-per-day sequence, backing the `SEQ4` portion of `tokenId` above |

Write-only via a client transaction at ticket-creation time; never read back
by the app (see `firestore.rules`).
